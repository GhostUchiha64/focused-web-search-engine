How It Works

Seed URL

User provides a starting URL and max page limit (up to 500)

Crawl

Spider fetches pages via cURL using BFS or DFS, stays within URL prefix

Index

Extracts titles, meta tags, links. Tokenizes terms with stopword removal

Store

Saves pages and terms to MySQL via PDO with SSL on Azure

Search

Keyword query ranked by number of title term hits descending

Features

Focused Crawler

Stays within the URL prefix. Ignores images, PDFs, binaries, and fragments automatically.

BFS & DFS Modes

Switch between breadth-first (SplQueue) and depth-first (SplStack) traversal strategies.

Stopword Removal

Common English words filtered before indexing for cleaner, more relevant results.

Ranked Results

Search results ordered by keyword hit count — most relevant pages appear first.

Politeness Delay

Configurable millisecond delay between requests to avoid hammering target servers.

Secure DB Connection

PDO with SSL to Azure MySQL. Preflight TCP check before every connection attempt.

Database Schema

pages

  • id (PK)
  • url (UNIQUE)
  • title
  • meta_keywords
  • meta_description
  • crawled_at

terms

  • id (PK)
  • term (UNIQUE)

page_terms

  • page_id (PK, FK)
  • term_id (PK, FK)
  • → pages.id
  • → terms.id

Tech Stack

PHP 8 cURL DOMDocument / XPath PDO (MySQL) Azure MySQL SplQueue / SplStack Apache CSS3
